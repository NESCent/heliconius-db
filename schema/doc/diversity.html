create table biotype
Biotype is essentially a named container of species. Typically, this will be the species that hybridized to give rise to a hybrid individual, and the name will usually be the species part of the binomial names, concatenated by "x". The biotype may also be used to designate race, beyond just a species; there may not be a taxonomic node for this, for example for races defined by host preference.
Field Name Data Type Size Default Value Other Foreign Key
biotype_id integer 11 PRIMARY KEY, NOT NULL
name VARCHAR 64 UNIQUE, NOT NULL, The unique name of the biotype. Typically the name of a species hybrid, with for example the species parts of the binomial names concatenated by "x".

Constraints

Type Fields
NOT NULL biotype_id
NOT NULL name
UNIQUE name

table individual_biotype
Designation of one or more possible biotypes to an individual. There should be more than one designation for an individual iff none of the designations are certain, not if the individual is a hybrid (use the biotype itself to represent the hybrid). Individuals raised from stocks have an implicit biotype (that of the stock), and do not need an explicit biotype designation.
Field Name Data Type Size Default Value Other Foreign Key
individual_biotype_id integer 11 PRIMARY KEY, NOT NULL
individual_id integer 10 UNIQUE, NOT NULL, The individual to which the biotype is being designated. individual.individual_id
biotype_id integer 10 UNIQUE, NOT NULL, The biotype being designated to the individual. biotype.biotype_id
certainty_type_id integer 10 UNIQUE, NOT NULL, The certainty of the designation, as a term from a controlled vocabulary. There cannot be more than one biotype designation if the designation is to be considered certain. cvterm.cvterm_id

Constraints

Type Fields

FOREIGN KEY individual_id
NOT NULL biotype_id
